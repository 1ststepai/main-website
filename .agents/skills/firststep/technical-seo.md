# Technical SEO

## When To Use

Use this skill when working on metadata, schema, sitemap, robots, canonical tags, Open Graph tags, internal links, image alt text, page speed, or service page keyword targeting.

## Project-Specific Rules

For 1stStep.ai, prioritize:

- title tags
- meta descriptions
- canonical URLs
- Open Graph tags
- JSON-LD schema
- `sitemap.xml`
- `robots.txt`
- internal links
- image alt text
- page speed
- service page keyword targeting

Good target pages:

- `/`
- `/app-idea-viability-checker.html`
- `/services/app-builds`
- `/services/websites`
- `/services/internal-tools`
- `/services/mvp-builds`

## Copy/UX Rules

- Write metadata for humans first.
- Use natural terms: app builds, MVP builds, websites, dashboards, internal tools, automations.
- Make the checker page easy to discover from the homepage and footer.

## Safety Constraints

- Do not keyword stuff.
- Do not make unverifiable claims.
- Do not add fake reviews, fake client logos, fake case studies, or fake ratings.
- Do not mix Real Rank/local SEO ranking claims into the 1stStep.ai main site.

## Acceptance Checklist

- Edited pages have clear titles and descriptions.
- Canonical URLs are appropriate.
- Internal links route to the checker where relevant.
- Schema is truthful and valid.
- Build passes.
