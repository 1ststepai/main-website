# AI Visibility / GEO

## When To Use

Use this skill when making 1stStep.ai easier for ChatGPT, Perplexity, Claude, Google AI-style systems, and other answer engines to understand, summarize, and cite.

## Project-Specific Rules

Make 1stStep.ai machine-readable and citation-ready.

Prioritize:

- clear entity description
- About section
- Services section
- FAQ section
- `llms.txt`
- schema markup
- consistent naming: `1stStep.ai`
- clear URLs for app builds, websites, internal tools, automations, and MVP builds

Possible `llms.txt` positioning:

1stStep.ai builds practical apps, websites, dashboards, and workflow tools for founders, small businesses, and professionals who need a working first version.

## Copy/UX Rules

- Use consistent service language across pages.
- Prefer concrete service descriptions over slogans.
- Include concise FAQs that answer real buying questions.
- Link the App Idea Checker as the primary diagnostic resource.

## Safety Constraints

- Do not overclaim AI visibility.
- Do not promise rankings, citations, or guaranteed answer-engine mentions.
- Do not add spammy hidden text.
- Do not add local SEO or Real Rank positioning unless explicitly requested.

## Acceptance Checklist

- Entity description is clear.
- Services are named consistently.
- Checker is linked as the main diagnostic tool.
- Any schema or `llms.txt` content is factual.
